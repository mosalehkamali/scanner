# Persian Accounting & Smart Barcode POS System

## Project Overview

Build a modern Persian-language web application that combines Accounting, Inventory Management, and Point of Sale (POS) functionality.

The main competitive advantage of this system is eliminating the need for expensive barcode scanner hardware. Instead, product barcode scanning must be performed using the smartphone camera with maximum speed, accuracy, and reliability.

The application must be designed for small shops, supermarkets, convenience stores, and retail businesses that need a simple, fast, and affordable accounting solution.

# Subscription System

## Available Plans

* 1 Month
* 3 Months
* 6 Months

### Registration Flow

1. User registers.
2. User selects a subscription plan.
3. User uploads a bank transfer receipt.
4. Admin reviews the receipt.
5. Admin approves the payment.
6. User gains access for the selected duration.

### Admin Controls

Admin can:

* Set prices for all plans
* Change prices anytime
* Configure destination bank card number
* Enable/disable plans

# POS / Invoice System

This is the most important module.

The POS interface must be extremely fast and simple.

## Product Adding Methods

### Method 1: Barcode Scanning

Use smartphone camera.

Requirements:

* Instant barcode detection
* Continuous scanning mode
* Auto-focus support
* Real-time recognition
* No page refresh
* Mobile optimized
* High accuracy
* Works in low-light environments
* Fast scan processing

### Method 2: Product Search

Some products may not have barcodes.

Cashier must be able to:

* Search products instantly
* Add products to invoice
* Filter search results in real-time

### Method 3: Manual Price Entry

Some items may not exist in the product database.

Cashier must be able to:

* Enter custom item name
* Enter custom price
* Add directly to invoice

---

# Multi-Invoice System

This feature is critical.

A cashier may serve multiple customers simultaneously.

The system must allow:

* Unlimited open invoices
* Switch between invoices instantly
* Save invoice state automatically
* Continue editing any invoice later

Example:

Customer A invoice → Open

Customer B arrives → Open second invoice

Customer C arrives → Open third invoice

Cashier can switch between all invoices without closing any of them.
